### 0.10 Deploy + branch protocol (CRITICAL)

**This repo deploys via GitHub Pages on `master` push.** A merge to `master` is a public deploy in seconds. With 100 agents fanning out, default-to-master would publish broken intermediate states to production.

**Required:**
1. **Never push to `master`.** Master is human-gate-only.
2. Each WU works on a branch named `swarm/<wu_id>` (e.g. `swarm/wu-form-01`). Branch off `master`.
3. After acceptance tests pass locally, the agent commits + pushes the branch + opens a draft PR against `master` titled `[<wu_id>] <one-line summary>`. PR body = the agent's full §0.5 YAML output.
4. **Agents do NOT merge.** A human (Justin or a designated merge-gate agent) reviews and merges. Set up branch protection on `master` (manual or via the GitHub API in WU-INFRA-00) to enforce.
5. **Local-first verification.** All acceptance tests using `curl` against URLs MUST first run against `python3 -m http.server 8080` (local) before anything reaches `master`. Production `curl` is post-deploy verification only and runs in a separate WU-VERIFY follow-up.
6. **Conflict policy.** If `git merge master` into the branch produces conflicts, the agent does NOT auto-resolve — it returns `status=needs_review` with the conflict list.
7. **Commit message format:** `<wu_id>: <imperative summary>` (no Conventional Commits prefix needed; keep brand voice). Body lists files changed and acceptance test results.
8. **Hooks:** never use `--no-verify`. If a hook fails, fix the underlying issue and create a new commit (never `--amend`).

#### 4.3.2 Standard form submit pattern (post WU-FORM-01)

This implementation lives in `/assets/rms-form.js` (created by WU-FORM-01). Each form's inline handler is a 4-line wrapper that calls `window.rmsSubmit(formId, ...)`.

```js
// /assets/rms-form.js — single source of truth for all four forms.
// FORMSPREE_ENDPOINT is injected at build time (or read from a meta tag set by Justin in WU-FORM-00).
// We read it from <meta name="rms-formspree" content="..."> rather than hard-coding so agents can't accidentally invent it.

const APPS_SCRIPT_ENDPOINT = 'https://script.google.com/macros/s/AKfycbx-ObsqmyFUYUeFsQjVri4lt42scvHeztXXDU43PCxDbnLeuc1RLZEhW1ME21V2YBUXCQ/exec';

function getFormspreeEndpoint() {
  const m = document.querySelector('meta[name="rms-formspree"]');
  return m && m.content && m.content !== '__SET_ME__' ? m.content : null;
}

window.rmsSubmit = async function rmsSubmit(formId, fd, statusEl, btn) {
  btn.disabled = true;
  btn.dataset.original = btn.textContent;
  btn.textContent = 'Sending…';
  statusEl.style.display = 'none';

  // 1) Fire Apps Script first as fire-and-forget — Justin's Sheet keeps receiving even if Formspree is down.
  fetch(APPS_SCRIPT_ENDPOINT, { method: 'POST', mode: 'no-cors', body: fd }).catch(() => {});

  // 2) Verifiable channel: Formspree. UI status is driven by this response only.
  const endpoint = getFormspreeEndpoint();
  if (!endpoint) {
    statusEl.textContent = 'Form temporarily unavailable. Please call (360) 644-4820 or email joberg@rivermountainsystems.com.';
    statusEl.style.color = '#ef4444';
    statusEl.style.display = 'block';
    btn.textContent = btn.dataset.original;
    btn.disabled = false;
    return;
  }

  const ctrl = new AbortController();
  const timeout = setTimeout(() => ctrl.abort(), 5000);
  try {
    const fsResp = await fetch(endpoint, {
      method: 'POST',
      body: fd,
      signal: ctrl.signal,
      headers: { 'Accept': 'application/json' }
    });
    clearTimeout(timeout);
    if (!fsResp.ok) throw new Error('formspree-' + fsResp.status);
    if (window.rmsTrack) window.rmsTrack('form_submit', { form: formId });
    statusEl.textContent = '✓ Got it — Justin will reach out within 1 business day. For urgent issues call (360) 644-4820.';
    statusEl.style.color = '#22c55e';
  } catch (err) {
    statusEl.textContent = 'Something went wrong sending that. Please call (360) 644-4820 or email joberg@rivermountainsystems.com.';
    statusEl.style.color = '#ef4444';
  } finally {
    statusEl.style.display = 'block';
    btn.textContent = btn.dataset.original;
    btn.disabled = false;
  }
};
```

**Page-side wiring (per form):**
```html
<!-- in <head> of every page using forms -->
<meta name="rms-formspree" content="__SET_ME__"><!-- WU-FORM-00 fills this from .swarm/secrets.local.md at deploy time -->
<script src="/assets/rms-form.js" defer></script>

<!-- inline submit handler — minimal, just collects fields and delegates -->
<script>
document.getElementById('quickForm').addEventListener('submit', function(e){
  e.preventDefault();
  var fd = new FormData(e.target);
  fd.append('source', 'hero_quickform'); // discriminator per form
  window.rmsSubmit('hero_quickform', fd, document.getElementById('quickStatus'), document.getElementById('quickSubmitBtn'));
});
</script>
```

**Why a `<meta>` tag for the endpoint:** keeps the secret out of agent context (agents can't see `secrets.local.md` content; only WU-FORM-00 / a deploy step injects it into the meta). Prevents the `__SET_ME__` placeholder from ever shipping live (WU-VERIFY-01 greps for `__SET_ME__` on production HTML).

#### 4.3.3 Tracking call convention
`rmsTrack('<verb>_<noun>', { …meta })` — `verb` past tense or imperative noun: `phone_click`, `form_submit`, `cta_click`, `cal_book`, `email_click`, `404_view`. `meta` always includes `location` for click events.
